## Abstract

This document evaluates **MemPalace** — the open-source AI memory system at `github.com/MemPalace/mempalace`, taken at HEAD on 2026-06-24 (release **v3.5.0**, MIT) [1] — and compares it against the Transpara-AI Civilization architecture. It is the fourth in the external-landscape evaluation series, applying the same pattern used for Sakana AI [20], the Hermes family [21], and the Google Open Knowledge Format [22] to a different subject: a working memory **store**. Unlike the prior MemPalace treatment — an investigation-tier wiki stub that explicitly carried the project's claims as *"upstream-asserted and unverified"* and never read the code [17] — this evaluation cloned the repository, read it as ground truth, and **recomputed every headline benchmark number from the committed per-question result files**.

The central finding parallels the Hermes and OKF results at the memory layer. MemPalace is, almost feature for feature, **the verbatim recall substrate the Civilization's own doctrine already names as a candidate** — `DF-V3.9-SPEC-006` states verbatim that *"MemPalace or equivalent recall storage may hold verbatim or near-verbatim prior context. It is advisory. It is not EventGraph, policy, certification evidence, or authority"* [10]. And it is a genuinely strong one: a local-first, byte-verbatim store with a clever lossy *index* that points back to lossless content, a hybrid (vector + BM25) retriever running on **local ONNX embeddings with zero API calls by default**, a temporal SQLite knowledge graph, a 35-tool MCP server, and the broadest, most reproducible **retrieval-recall** benchmark suite in its field [1][2][3].

But MemPalace is a deliberately **fail-open recall layer**: it stores what was said and returns what is similar; it has no notion of *trust*, no record of how a memory *influenced a decision*, no quarantine that blocks *use*, and its per-project and per-agent scoping is a soft metadata convention any caller can omit [1][2]. That permissiveness is *correct* for an advisory recall store. The Civilization, by contrast, wraps the same recall function in a **fail-closed influence-accounting model**: any memory that materially influences factory work must be emitted as a `MemoryReference` carrying a trust level, a `source_hash`, a `redaction_state`, a `freshness_status`, contradiction references, and an `influence_summary` — *before certification can pass* [10][11].

The one-line result: **MemPalace retrieves what was said; the Civilization governs what a memory is allowed to change — and refuses to let it change anything off the record.** A separate, load-bearing finding qualifies the marketing: MemPalace's "**best-benchmarked**" claim is real but narrow. Every headline number is **retrieval recall@K** (is the gold session in the top-K retrieved?), *not* end-to-end QA answer accuracy — the repository contains no answer-accuracy harness — and **no competitor is ever run** (rival figures are quoted, cross-metric, from their own posts) [3]. Six concrete, implementable learning opportunities follow, all consistent with the standing rule that external frameworks stay outside the control core [10][21]: they describe how to *adopt MemPalace as the recall adapter behind the `MemoryReference` boundary*, never as truth, authority, or certification evidence.

## 2. MemPalace: Capability Overview

MemPalace describes itself as *"local-first AI memory. Verbatim storage, pluggable backend, 96.6% R@5 raw on LongMemEval — zero API calls"* [1]. It is a Python application distributed on PyPI as `mempalace`, run as a CLI, an MCP server, or a long-lived local daemon, and wired into coding agents (Claude Code, Codex, Cursor, Antigravity, OpenClaw) through editor plugins and auto-save hooks [1][2]. Its thesis is a memory store that *"treats every word you have shared as sacred, stores it verbatim … We never summarize. We never paraphrase. We return your exact words"* [1].

### 2.1 The Memory-Palace Model (Code-Anchored)

The "memory palace" metaphor is a **set of metadata fields on records**, not a tree of objects on disk [1]. A **drawer** is the unit of stored content — a verbatim text chunk persisted as a document in a vector collection (ChromaDB by default), with a deterministic ID and a metadata dict carrying `wing`, `room`, `hall`, `entities`, `source_file`, `filed_at`, and line ranges (`miner._build_drawer_metadata`). A **wing** (people/projects) and a **room** (topic grouping) are simply slugs in that metadata; a **hall** is a content-type tag; **tunnels** (cross-wing links) and **hallways** (within-wing entity links) live in sibling `tunnels.json` / `hallways.json` graph files, not in the vector store.

Two properties are worth stating precisely because they cut against a casual reading of the marketing:

- **Verbatim is true at the byte level, but the corpus is chunked.** Text is stored without summarization or paraphrase across every ingest path (project files, conversation transcripts, even the "extract" modes select *verbatim spans* rather than rewriting) — this is code-verified. But content is segmented into overlapping ~800-character drawers and re-stitched with neighbors at retrieval time; "verbatim" describes fidelity, not whole-document storage [1].
- **The one lossy layer is an index that points to the lossless store.** MemPalace builds "closets" in a compact symbolic format (AAAK) whose own header declares it *"a lossy summarization format … The original text cannot be reconstructed from AAAK output … It is a structured summary layer (closets) that points to the original verbatim content (drawers)"* [1]. Closets are a **ranking signal, never a gate** — they boost retrieval but never replace the returned verbatim content. This is an elegant inversion: a lossy index over a lossless store.

> ⚠ **Doc-vs-code note.** The README/CLAUDE.md framing "rooms = days/sessions" is **contradicted by code**: room routing is topical (folder/keyword), not temporal. The README's `mempalace_list_agents` "discoverable at runtime" claim has **no implementing MCP tool** at this HEAD. Per-agent "wings and diaries" are a **metadata convention**, not an enforced isolation boundary. These are noted as ground-truth corrections, not as faults.

### 2.2 Retrieval (Code-Anchored)

The served default is **hybrid retrieval**, not pure vector search [1]. `searcher.search_memories` over-fetches vector candidates from the drawers collection (the "floor"), applies a closet rank-boost, optionally unions in lexical (BM25) candidates, then re-ranks with a blended score `0.6·vector_similarity + 0.4·Okapi-BM25`. Embeddings are **local ONNX**: the default model is `all-MiniLM-L6-v2` (384-dim), with `embeddinggemma-300m` (multilingual, MRL-truncated to 384-dim) as a drop-in alternative — both run in-process via ONNX Runtime with **no embedding API call** [2]. A vector-disabled fallback reads the Chroma SQLite directly via FTS5 when the HNSW index is corrupt.

> ⚠ **Evidence note.** An **LLM reranker** appears in the *benchmark* harness (and in README figures), but it is **not wired into the served `search_memories` path**; the default retrieval is vector + BM25 + closet boost with no LLM in the loop [1][3]. Treat "Hybrid + LLM rerank" as a benchmark configuration, not the shipped default.

### 2.3 Temporal Knowledge Graph (Code-Anchored)

MemPalace ships a temporal entity-relationship graph backed by **local SQLite** — its source comments position it explicitly as *"what competes with Zep's temporal knowledge graph … We use SQLite locally (free)"* [1]. Triples (`subject → predicate → object`) carry `valid_from` / `valid_to` validity windows; a fact is current iff `valid_to IS NULL`; `invalidate()` closes the interval rather than deleting (an audit-friendly choice); and as-of queries filter on the window. It is a **plain SQL adjacency model** (valid-time only, no graph engine, no multi-hop traversal in the KG itself) — capable and honest, but not a graph database.

### 2.4 Surfaces: MCP, Backends, Integrations (Code-Anchored)

- **MCP server — 35 tools** (verified by enumerating the central `TOOLS` dict and cross-checking the docs; the repository's own test pins its README count to the live code) [2]. The prior wiki article's "**29 tools**" is **stale** [17]. Tools span status, taxonomy, search, drawer CRUD, mining/sync, the knowledge graph (5 tools), graph/tunnel navigation (9), and agent diaries. Argument allowlisting rejects spoofed internal parameters.
- **Four pluggable backends** behind an RFC-001 contract: `chroma` (default, local), `sqlite_exact` (local exact brute-force), `pgvector` (Postgres), `qdrant` (Qdrant) [2]. The backend contract enforces a hard **per-`PalaceRef` isolation guarantee** ("a record written for one palace MUST NOT be returned, modified, or deleted by an operation for a different palace") — the one place MemPalace is genuinely fail-closed.
- **Editor integrations + auto-save hooks** for Claude Code, Codex, Cursor, and Antigravity, plus an OpenClaw skill; hooks capture verbatim transcripts on `Stop` / `SessionEnd` / `PreCompact` and inject recall on wake [2]. A shared "search before answering" recall protocol is the cross-harness contract.

### 2.5 Privacy, License, and Maturity

- **Privacy by architecture (code-verified).** With the default Chroma/SQLite backends, the core path (embed → store → search) has **no external egress**; no telemetry call-sites exist in `mempalace/`; the MCP server even redirects dependency stdout to protect the stdio protocol [2]. Two honest caveats: a one-time embedding-model download from HuggingFace on first run (an **air-gap consideration**), and an opt-in BYOK LLM-refinement path that is **on by default pointed at localhost Ollama** (it degrades silently if absent and never reaches an external endpoint without an explicitly configured provider behind a local-vs-external consent gate) [2].
- **License & supply chain.** MIT, with a permissive default dependency tree (ChromaDB Apache-2.0, NumPy BSD, etc.); the only copyleft item is `psycopg` (LGPL-3.0), confined to the optional `[pgvector]` extra; the MCP server is hand-rolled (no SDK dependency), shrinking the surface [4]. The README opens with a **"Scam alert"**: the only official sources are the GitHub repo, the PyPI `mempalace` package, and `mempalaceofficial.com`; *"any other domain (including `.tech`, `.net`, or other `.com` variants) is an impostor and may distribute malware"* [1]. A maintained `docs/HISTORY.md` ledger records benchmark-claim retractions and the impostor-domain timeline — a notable transparency signal.
- **Maturity.** ~2.5 months old (created 2026-04-05), **56k+ stars / 7k+ forks**, ~30 contributors concentrated in a small pseudonymous core, **Development Status :: 4 — Beta**, very high velocity (1,800+ PRs, 30+ releases in ~11 weeks), ~3,000 test functions behind an 85% coverage gate [4]. The profile is *explosive adoption of a young, still-self-correcting project*, not a battle-tested one — which is exactly why the crosswalk's license-and-supply-chain gate matters before any integration [18].

### 2.6 The "Best-Benchmarked" Claim (Artifact-Anchored)

MemPalace's headline positioning is *"the best-benchmarked open-source AI memory system."* Reading the harness and recomputing the aggregates from the committed per-question files yields a precise, two-sided picture [3].

**The numbers reproduce — and they measure retrieval recall, not answer accuracy.** Every headline figure is **recall@K** ("is the gold session/evidence inside the top-K retrieved candidates?"), and the repository contains **no end-to-end QA-accuracy harness** (it is listed as planned, not built) [3]:

| Headline claim | Recomputed value | What it actually measures | Reproducible? |
|---|---|---|---|
| "96.6% R@5 raw on LongMemEval — zero API calls" | **0.966 R@5** (500 q) | Session-level **retrieval recall@5**, local embeddings, zero API | **Yes** — exact, deterministic |
| Hybrid v4, held-out 450 q | **0.984 R@5** | Retrieval recall@5 on a clean held-out split (seed 42, 0 dev-set contamination) | **Yes** — exact; the honest generalizable number |
| Hybrid v4 + LLM rerank, full 500 | **0.992 R@5** (README "≥99%") | Recall@5 after an LLM promotes one of the top-20 | Frozen artifact verified; live rerank non-deterministic |
| LoCoMo, raw / hybrid v5, top-10 | **0.603 / 0.889 R@10** | Evidence-**session** recall@10 | **Yes** — exact |
| ConvoMem, 250 items | **0.929** | Fuzzy-substring **recall**@10 | **Yes** — exact |
| MemBench, 8,500 items | **0.803 hit@5** | Target-turn **retrieval recall**@5 | **Yes** — exact, accurately described |

**Honesty signals (genuine).** The held-out split is real and uncontaminated; the project publishes the *lower* held-out 98.4% as its lead hybrid figure; `BENCHMARKS.md` self-discloses that three hybrid-v4 boosts were reverse-engineered from specific failing questions and labels it *"teaching to the test … a significant methodological problem"*; and the public README/website have removed the old cross-system table and now state that recall ≠ accuracy [3]. This auditability — committed per-question files, deterministic zero-API path — is real and better than most of the field.

**Overclaims (residual, internal).** The internal `BENCHMARKS.md` still asserts a "**100%** … across all 500" LongMemEval result whose committed file is **only 20 questions** (the real full-500 value is 99.2%), and a LoCoMo "100%" that is **uncommitted and structurally trivial** (top-50 retrieval over conversations with ≤32 sessions guarantees every session is "retrieved") — a result the authors themselves disavow; ConvoMem's "75K+ QA pairs" framing reflects **250 items actually run across 5 of 6 categories** [3]. And **no competitor is run in-repo** — every rival number (Mem0, Zep, etc.) is quoted from the competitor's own post, comparing MemPalace **recall** against competitors' **accuracy** [3].

**Competitive context — is "best-benchmarked" defensible? (Third-party-cited.)** It is **contested, bordering on unfalsifiable** [23][24][25]. The field's two reference benchmarks are scored on **end-to-end QA answer accuracy** (LLM-judged), with retrieval recall a *diagnostic*, not the headline: LongMemEval (ICLR 2025) reports QA accuracy across six question types over ~115k-token (`-S`) to ~500-session (`-M`) histories [23]; LoCoMo reports QA accuracy over very long conversations and is widely criticized for a small N=10 corpus, flawed gold answers, and saturation — a filesystem-only baseline scores ~74% [24]. MemPalace's headline is **retrieval recall@5**, a strictly easier question, and the repository has **no end-to-end accuracy number at all** — so on the benchmarks' *primary* metric MemPalace is literally unranked. Competitors lead with accuracy:

| System | Approach | Benchmark | Headline | Metric |
|---|---|---|---|---|
| **MemPalace** | Verbatim store + hybrid recall | LongMemEval | 96.6% R@5 | **Retrieval recall** |
| Mem0 | Vector memory layer | LoCoMo | 66.9% (+26% vs OpenAI memory) | QA accuracy (LLM-judge) |
| Zep / Graphiti | Temporal knowledge graph | LongMemEval | 71.2% (vs 63.8% full-context) | QA accuracy |
| Letta / MemGPT | Agentic self-editing | LoCoMo | 74.0% (plain filesystem) | QA accuracy |
| Supermemory | Retrieval swarm | LongMemEval-S | ~85% prod / ~99% experimental | QA accuracy |
| Mastra | Observational-memory module | LongMemEval | 84–95% | QA accuracy |

There is also **no neutral referee**: every score is vendor-self-reported on a self-chosen harness, the two loudest claimants (Mem0, Zep) have **publicly invalidated each other's numbers**, and independent reproductions land materially lower (Mem0's self-cited ~91.6% LoCoMo reproduces nearer 58–66%) [25]. "Best-benchmarked" is therefore defensible only as a narrow **process/transparency** claim — *"ran the broadest suite, disclosed methodology most fully,"* which MemPalace's committed-per-question files genuinely support — and **not** as a **performance-superiority** claim, because the recall figure it leads with cannot be compared to the accuracy figures the field ranks on. This is exactly why §5.5 treats a recall number as a necessary diagnostic, never a sufficient quality claim.

> ⚠ **Evidence note.** §2.6's MemPalace numbers are **artifact-anchored** (recomputed from the repository's committed per-question files). The recall-vs-accuracy framing and the competitor figures are **third-party-cited** (vendor self-reports on differing harnesses) and carried at lower confidence [23][24][25].

## 3. Transpara-AI Civilization: Baseline Architecture

The Transpara-AI Civilization is a self-organizing AI agent civilisation built on the EventGraph substrate. It is not a product — it is a platform for collective existence whose goal is to build products that fund the agents that build more products, on a transparent, auditable causal chain. (This section reuses the TAI-RES-2026-001/-002/-003 baseline [19] so the four evaluations stay parallel, and foregrounds the **Memory/Knowledge model**, which is the native machinery most relevant to a memory-store comparison.)

### 3.1 Architecture Layers

| Layer | Description |
|---|---|
| EventGraph Substrate | 201 primitives. Causal chain where every action, resource, and outcome is linked. Trust model, authority levels (Required → Recommended → Notification), cryptographic consent. All on Postgres on-prem. |
| Hive Runtime | 4 bootstrap agents: CTO, Guardian, Researcher, Planner/Implementer. FSM-based agentic loop. Go binary. Deterministic agent identity. |
| Social Grammar | Constitutional law for agents. 10 invariants. 8 formal agent rights (existence, **memory**, identity, communication, purpose, dignity, transparency, boundaries). Dual human-agent consent to amend. |
| **Memory / Knowledge Model** | Governed advisory recall and curated knowledge. Any memory/knowledge that *materially influences* factory work must be emitted as a `MemoryReference`/`KnowledgeReference` with trust level, `influence_summary`, and redaction state *before certification can pass*. Detailed in §3.3. |
| Decision Tree Evolution | Agents learn within their lifetime: after ≥50 hits at ≥95% accuracy, deterministic branches replace LLM calls. |
| Capability Evolution Chain | Governed self-improvement of shared capabilities — the subject of the Hermes evaluation [21]. Ten typed records, code-enforced leash. |
| 13 Product Layers | Work, Market, Social, Justice, Build, **Knowledge**, Alignment, Identity, Bond, Belonging, Meaning, Evolution, Being. Layer 6 (Knowledge) is the product home of curated knowledge. |
| Self-Sustaining Economy | Revenue funds agents; agents build products; products generate revenue. Every resource on the causal chain, publicly auditable. |

### 3.2 The Advisory Memory/Knowledge Layer — Where MemPalace Already Has a Slot

The Civilization has a designed slot for a recall store, and the doctrine names MemPalace in it. The advisory-layer article frames the layer as the part that lets prior context *"inform factory work without ever governing it … it is not EventGraph, not policy, not certification evidence, and not authority"* [12]. The doctrine is more specific still: *"**Memory (MemPalace or equivalent recall storage)** holds verbatim or near-verbatim prior context. It is advisory; it is not EventGraph, policy, certification evidence, or authority"* [10][12]. Critically, this layer is **specified but not yet exercised in the proven control plane**: Base Slice 0 *"explicitly forbids memory recall and knowledge reuse"* [10][13]. The receiving socket exists in doctrine and code; it has not been switched on.

### 3.3 The Memory/Knowledge Model (the native counterpart of a MemPalace recall)

Where MemPalace defines a *retrieved document*, the Civilization defines a *governed reference*. The doctrine (`DF-V3.9-SPEC-006`, accepted/canonical) is explicit [10]:

> *"MemoryReference must be created when memory materially influences planning, generation, repair, release review, or capability evolution … If advisory memory or knowledge materially affects factory work, the system must emit a MemoryReference or KnowledgeReference with `influence_summary` before certification can pass."*

The minimum influence-reference field set is **thirteen**, not free-form [10]:

```text
reference_id · source_system · source_ref · source_hash/immutable locator ·
retrieved_at · used_by_actor · used_in_task · influence_summary · risk_scope ·
trust_level · freshness_status · redaction_state · contradiction_refs
```

Memory carries a **trust level** drawn from a fixed lattice — `raw · user_confirmed · system_observed · reviewed · deprecated · quarantined` — and the doctrine is fail-closed about secrets: *"Secret-bearing memory must be quarantined. Quarantined memory cannot be used"* [10]. Both reference kinds are first-class EventGraph types, defined in code as `TypeMemoryReference` (schema.go L40) and `TypeKnowledgeReference` (L41), alongside `MemoryIngested`/`MemoryIndexed`/`MemoryScopeAssigned`/`MemoryRedactionApplied` lifecycle events (L43–46) [11]. The fail-closed posture is enforced in code at the store boundary: secret indicators force `classification: secret → redaction_state: quarantined`, which makes the memory both un-retrievable and un-usable; an open high/critical `ContradictionLog` blocks high-risk use; stale knowledge is blocked for high-risk planning [11]. A memory artifact in the Civilization is therefore not a document that *describes* prior context; it is a causally-linked record that *accounts for how that memory influenced a decision*, gated on trust, freshness, contradiction, and redaction.

> ⚠ **Fail-legible note.** One honest asymmetry: the six-value trust lattice is a **doctrine** constraint; the Go validator currently enforces it as a *denylist* (it blocks `quarantined`/`stale` and requires `trust_level` non-empty) rather than an allowlist of the six values [11]. By the Civilization's own fail-closed-by-default standard, that is a gap to close (allowlist the lattice; deny the unknown) — and it is precisely the kind of place a recall integration must not widen.

## 4. Comparative Analysis

The central finding is that MemPalace and the Civilization are **complementary at the memory layer they appear to share** — but they sit on opposite sides of a single clean seam: MemPalace owns *storage and retrieval* and is fail-open by design; the Civilization owns *trust, influence, and accountability* and is fail-closed by design. MemPalace is, almost feature for feature, the advisory recall store the doctrine specifies as a candidate — the way OKF was a serialization of the wiki [22] and Hermes a reference implementation of the optimizer loop [21].

### 4.1 Capability Comparison Matrix

| Dimension | MemPalace (v3.5.0) | Transpara-AI Civilization | Verdict |
|---|---|---|---|
| Primary artifact | A local recall **store** (verbatim drawers + hybrid retrieval) | A governed **influence reference** (`MemoryReference` on EventGraph) | Different layers |
| Storage shape | Verbatim chunks in a vector store + SQLite KG | Verbatim/near-verbatim recall **+** typed EventGraph reference records | **Same substrate; Civilization adds a governed layer** |
| Required structure | Free-form metadata; no mandatory trust fields | 13-field minimum influence reference + trust lattice | **Civilization vastly richer** |
| Failure posture | Fail-**open**: returns what is similar; soft scoping; no trust gate | Fail-**closed**: quarantine blocks use; gates default to deny | **Opposite by design (both correct in place)** |
| Trust / provenance | None — every well-formed memory is equally retrievable | `trust_level` lattice + `source_hash` + `freshness_status` | **Civilization unique** |
| Influence accountability | None — stores/retrieves, never records *how memory shaped a decision* | `influence_summary` required *before certification can pass* | **Civilization unique** |
| Secrets / redaction | No quarantine-blocks-*use*; relies on user scoping | Secret-bearing memory quarantined and unusable; `redaction_state` | **Civilization unique** |
| Contradiction handling | None | `contradiction_refs`; open high/critical blocks high-risk use | **Civilization deeper** |

| Retrieval engine | **Excellent** — local hybrid (vector+BM25+closet), reproducible recall suite | No best-in-class recall engine; advisory layer not yet exercised | **MemPalace leads** |
| Local-first / air-gap | **Excellent** — local ONNX, zero-API default, MIT | On-prem Postgres substrate; no drop-in local recall engine | **MemPalace leads** |
| Benchmark transparency | **Broad, reproducible, committed per-question files** (recall) | Internal derivation; no public memory benchmarks | **MemPalace leads** |
| Ecosystem / integrations | 35-tool MCP, 4 backends, 4 editor plugins, OpenClaw | Single-civilisation substrate; Researcher/MemoryKeeper roles | **MemPalace leads on reach** |

| Governance of the system | Pseudonymous small team, beta, rapid churn | Canonical, versioned, dual-consent constitutional change | Different goals |

### 4.2 Shared Territory

MemPalace and the Civilization converge on the same instincts — unsurprisingly, since the doctrine was written with "MemPalace or equivalent" as its mental model [10]:

- **Verbatim, not paraphrase.** MemPalace's *"never summarize, never paraphrase"* [1] is the doctrine's *"verbatim or near-verbatim prior context"* [10]. Both refuse lossy compression of the source.
- **Advisory, not authority.** MemPalace's diaries and recall are explicitly advisory context for an agent; the Civilization's memory layer *"advise[s] but do[es] not govern"* [10][12]. Same boundary, named identically.
- **Local-first, plain stores.** MemPalace's local ONNX + SQLite/Chroma [2] mirrors the Civilization's on-prem-Postgres, no-cloud-dependency stance. Both rejected the proprietary-SaaS-memory path.
- **Temporal / as-of recall.** MemPalace's `valid_from/valid_to` invalidate-don't-delete KG [1] is the same instinct as the doctrine's freshness windows and as-of intent [10].
- **Agents curate their own memory.** MemPalace's per-agent diaries and "search before answering" protocol [2] are the Civilization's Researcher/MemoryKeeper loop in different words.

### 4.3 The Civilization's Structural Advantages

These have no MemPalace equivalent and represent the deepest difference — every one of them lives in the *governance* half of the storage/governance seam:

- **Trust is a property of the record, not the retriever's optimism.** MemPalace has no trust field; a consumer must trust every well-formed drawer equally. The Civilization grades every reference on a six-level lattice and refuses to use `quarantined` memory [10][11].
- **Influence is accounted for, not merely retrieved.** MemPalace returns documents; it never records how a memory *shaped* a decision. The Civilization requires an `influence_summary` linking memory to the decision it influenced, and **blocks certification** until that record exists [10]. This is the difference between a search box and an audit trail.
- **The failure posture is inverted — deliberately.** MemPalace is fail-open (returns nearest neighbours, soft scoping, no trust gate), which is right for a *recall* layer. The Civilization is fail-closed: secret-bearing, stale, or contradicted memory is refused at *use*, not waved through. Both are correct in their layer; the danger is only in confusing them — which is exactly the crosswalk's stated risk, *"memory can influence work without evidence if miswired"* [18] (see §5.1).
- **Causal accountability.** Every `MemoryReference` is a linked EventGraph record tied to the actor and task that used it [11]. MemPalace offers git history and a validity-windowed SQLite graph, not a causal chain.
- Plus the civilisation-wide advantages shared with the prior evaluations: constitutional agent rights (including a memory right), the self-sustaining economy, and 13-layer scope [20][21].

It does not reopen them; it **confirms them against the current released baseline and resolves the prior treatment's open "unverified" flags.**

## 5. Strategic Learning Opportunities

Six concrete, implementable opportunities. None requires new substrate or a reversal of any decision: each describes how to *adopt MemPalace as the recall adapter behind the `MemoryReference` boundary* — the EventGraph trust model remains the authority.

### 5.1 MemPalace → the Recall Substrate Behind the `MemoryReference` Boundary (realize ADR-0008)

**Current state:** ADR-0008 proposed MemPalace as the verbatim recall substrate but selected no implementation owner; Base Slice 0 forbids recall, so the receiving socket is unexercised [13][16].

**MemPalace pattern:** A local-first store that returns nearest-neighbour verbatim drawers with no trust, provenance, or influence semantics — fail-open by design [1][2].

**Learning:** This is the load-bearing, *fail-closed* learning. Wire MemPalace **behind an adapter that emits a governed `MemoryReference` for every recall that materially influences work**, before that work can be certified [10]. Concretely: recalled content enters at trust level **`raw`** and may rise no higher until reviewed; `source_hash`, `source_system`, `retrieved_at`, and `redaction_state` are recorded; secret indicators force quarantine (and quarantined memory cannot be used); and `influence_summary` is required before any gated decision. MemPalace's permissiveness must **stop at the trust boundary**: a soft-scoped, untrusted nearest neighbour is harmless as *recall* and unacceptable as an ungated input to *certification*. *(This is the MemPalace instance of the standing fail-closed-by-default principle: allowlist what may influence certified work; never let an unverified recall through because it was similar.)*

### 5.2 The 35-Tool MCP Server + Recall Protocol → the MemoryKeeper/Researcher Recall Interface (gated)

**Current state:** The Researcher/MemoryKeeper roles are specified to read sources and emit governed knowledge, but the "search prior context before acting" loop has no concrete recall API.

**MemPalace pattern:** A 35-tool MCP server (`search`, `kg_query`, `diary_read/write`, taxonomy) plus a cross-harness *"search before answering"* protocol [2].

**Learning:** Adopt MemPalace's **tool surface as the recall adapter's API**, but route every read that influences work through governed `MemoryReference` emission (§5.1). The MCP boundary is a clean adapter seam — reference-only adoption that captures MemPalace's ergonomics without importing its trust posture. Keep diaries advisory: an agent diary is `raw`-trust context, never certification evidence.

### 5.3 The Temporal Knowledge Graph → an Implementation Pattern for `freshness_status` and As-Of Recall

**Current state:** The doctrine specifies per-class freshness windows and as-of semantics, but the v39 Go package does **not compute staleness over time** — it checks a string flag [11].

**MemPalace pattern:** `valid_from/valid_to` triples with **invalidate-don't-delete** and as-of queries, in plain SQLite [1].

**Learning:** Use MemPalace's temporal-KG mechanics as the **concrete pattern** for realizing `freshness_status` and as-of recall — close intervals rather than delete (audit-friendly), compute staleness against the doctrine's windows, and surface contradictions as first-class records. Emit the results as governed `KnowledgeReference`s; do not adopt MemPalace's KG as a truth source.

### 5.4 Local-ONNX Zero-API Embeddings → the Air-Gap-Default Recall Engine

**Current state:** The Civilization's air-gap-default constraint has no selected local recall engine; the advisory layer has no embedding/retrieval implementation.

**MemPalace pattern:** Local ONNX embeddings (MiniLM / EmbeddingGemma), hybrid local retrieval, no API key, no egress by default [2].

**Learning:** MemPalace's all-local stack is an ideal fit for the **air-gap-default** infrastructure constraint — same instinct as OKF's self-contained viewer [22], but for the recall *engine*. One **fail-closed caveat**: the first-run model download from HuggingFace must be replaced by a **vendored, hash-pinned offline weight cache** before air-gapped use — an unmet network dependency is exactly the kind of silent gap the air-gap rule exists to catch.

### 5.5 The Reproducible Recall Benchmark Rig → a Governed `EvalDataset` for the Recall Adapter

**Current state:** The Capability Evolution chain requires an `EvalDataset` fixed before optimization, but the recall adapter has no quality harness [21].

**MemPalace pattern:** A broad, **reproducible, committed-per-question** retrieval-recall suite (LongMemEval, LoCoMo, MemBench) with a deterministic zero-API path [3].

**Learning:** Adopt MemPalace's benchmark rig as the **`EvalDataset` engine for the recall adapter** — but measure **governed-recall effect on the gated decision** (did the retrieved `MemoryReference` improve the outcome it influenced?), not merely R@K. Do **not** inherit the recall≠accuracy conflation that §2.6 flags: a recall number is a necessary diagnostic, not a sufficient quality claim. This keeps the Civilization's own memory evaluation honest where the field's marketing is not.

### 5.6 The Supply-Chain / Governance Posture → the Template for the Integration Packet's License & Supply-Chain Gate

**Current state:** The crosswalk requires a license + supply-chain review before any memory adapter is integrated [18]. A transpara-ai fork (`transpara-ai/mempalace`) already exists and currently **passively mirrors** upstream `main` (in sync at v3.5.0, 0 commits behind as of 2026-06-24) — but no concrete pin-and-review gate is specified, and passive mirroring of a fast-moving upstream is itself the risk.

**MemPalace pattern:** MIT and permissive by default, but a ~2.5-month-old, pseudonymously-maintained, high-churn beta with **active impostor domains that "may distribute malware"** [1][4].

**Learning:** Make MemPalace the worked example of the **fail-closed supply-chain gate**, and apply it to the fork that already exists. Stop passively tracking `main`; **pin the fork to a reviewed commit hash** (never "latest"); verify PyPI provenance against the official sources the project itself names (and treat the impostor `.tech`/`.net` domains as a live threat); confine the LGPL `psycopg` `[pgvector]` extra out of any redistributed default build; and treat each upstream sync as a fresh **`raw`-trust** input that must re-clear the gate before the memory-integration packet advances. The maturity profile — and the live impostor-malware domains — make this non-optional, not ceremonial.

## 6. The Inverse: What MemPalace Cannot Learn From the Civilization

| Civilization Capability | Why MemPalace Has No Equivalent |
|---|---|
| Influence accounting that gates certification | A MemPalace drawer *is* retrieved context; it never records how a memory *shaped a decision*. The Civilization blocks certification until an `influence_summary` exists [10]. This is a substrate property, not a metadata field. |
| Trust-graded memory | MemPalace retrieves every well-formed drawer equally. The Civilization grades memory on a six-level lattice and refuses to use `quarantined` records [10][11]. MemPalace cannot add this without ceasing to be a neutral recall store. |
| Fail-closed posture | MemPalace is fail-open by design (nearest-neighbour return, soft scoping) — correct for recall, wrong for trust. The Civilization defaults to deny. Inverting this would break MemPalace's job, which is precisely why the two belong at different layers. |
| Secret quarantine that blocks *use* | MemPalace relies on user scoping; a secret in a drawer is retrievable. The Civilization forces secret-bearing memory to `quarantined` and forbids its use at the store boundary [11]. |
| Contradiction blocking | MemPalace has no contradiction model. The Civilization keeps `contradiction_refs` as a first-class field and blocks open high/critical contradictions from high-risk use [10][11]. |
| Causal accountability | MemPalace offers git history and a validity-windowed SQLite graph. The Civilization links every `MemoryReference` to the actor and task that used it on the EventGraph causal chain [11]. |
| Constitutional agent rights (incl. a memory right) | MemPalace is a store; agents are users of it. The Civilization grants 8 formal rights, including a memory right, enforced architecturally [20][21]. |
| Self-sustaining economy & 13-layer scope | MemPalace is a free OSS store with no economic or societal model. The Civilization funds agents from the products agents build and spans Work through Being [20][21]. |

## 7. Conclusions

MemPalace is the most *functionally convergent* external system the Civilization has evaluated on the memory axis: it is, feature for feature, the **advisory recall store the doctrine already names as a candidate** [10], and a genuinely strong one — local-first, byte-verbatim, hybrid-retrieved on zero-API local embeddings, with a temporal SQLite graph, a 35-tool MCP surface, broad editor integration, and the most reproducible **retrieval-recall** benchmark suite in its field [1][2][3]. That convergence is not coincidence; the doctrine was written with "MemPalace or equivalent" as its mental model [10].

But MemPalace stops where a recall store should. It retrieves memory; it does not **govern memory's influence.** It has no trust grading, no influence accounting, no certification gate, no fail-closed posture at *use* — the entire `MemoryReference` layer. The difference between the two systems is therefore not the text on disk; it is **what a remembered fact is allowed to change.** MemPalace says: return what is similar, trust every well-formed drawer, scope softly. The Civilization says: a memory may influence certified work only with a trust level, a provenance hash, a redaction state, a freshness status, a contradiction check, and an influence summary — and certification waits until it has them. Both rules are correct in their own layer, and the whole strategic value lies in keeping them separate: **adopt MemPalace at the recall boundary, never at the governance core.** Separately, the "best-benchmarked" claim is real but narrow — reproducible **retrieval recall**, not QA accuracy, and never run head-to-head against a competitor [3]; it should be cited precisely, not inherited.

**Priority recommendations:**

1. **Highest priority:** Define the fail-closed `MemoryReference` import boundary (§5.1) *before* wiring any MemPalace recall into gated work — ingest at `raw`, quarantine secrets, require `influence_summary`. This is the one learning with downside risk if skipped, and it is a direct application of the fail-closed-by-default standard and the crosswalk's "miswired" risk [18].
2. **High priority:** Move **ADR-0008 from *Proposed* to *Accepted (with conditions)*** [16], conditioned on §5.1's boundary and §5.6's supply-chain gate, and adopt MemPalace's MCP tool surface as the recall adapter API (§5.2). This turns a stalled proposal into a governed, owned integration path.
3. **Medium priority:** Use MemPalace's temporal-KG mechanics as the implementation pattern for `freshness_status` / as-of recall (§5.3), and adopt its all-local stack as the air-gap-default recall engine behind a vendored, hash-pinned weight cache (§5.4).
4. **Medium priority:** Adopt the reproducible benchmark rig as a governed `EvalDataset` for the recall adapter — measuring governed-recall effect on the gated decision, not bare R@K (§5.5).
5. **Lower priority (gate first):** Execute §5.6's license + supply-chain gate (vendored commit pin, PyPI provenance, impostor-domain awareness, LGPL extra confinement) as the precondition for any of the above to touch code.

**Standing constraints reaffirmed:** Per Decision 15, MemPalace stays a recall adapter and pattern — never a truth source, authority, certification authority, or control-plane primitive [10][21]. MemPalace's fail-open recall is adopted only at the recall boundary; the trust boundary remains fail-closed. Recalled memory never enters above `raw` trust without review; secret-bearing memory is quarantined and unusable; and no integration proceeds before the supply-chain gate passes.
